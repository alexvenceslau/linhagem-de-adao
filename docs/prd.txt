# PRD - Linhagem Bíblica: De Adão a Cristo
**Product Requirements Document v1.0**

---

## 1. VISÃO GERAL DO PRODUTO

### 1.1 Resumo Executivo
Aplicação web interativa que visualiza a genealogia bíblica completa desde Adão até Jesus Cristo, integrando dados do Antigo e Novo Testamento em uma experiência visual educativa e devocional.

### 1.2 Problema a Resolver
- **Fragmentação**: Genealogias bíblicas estão dispersas em múltiplos livros (Gênesis, Mateus, Lucas)
- **Complexidade**: Idades, nomes e variantes textuais confundem leitores iniciantes
- **Desconexão**: Falta de visualização da continuidade histórica de Adão a Cristo
- **Engajamento**: Textos estáticos não promovem exploração interativa

### 1.3 Objetivos do Produto
**Principal**: Criar ferramenta visual que conecte narrativas bíblicas (criação → Messias)
**Secundários**:
- Facilitar estudos devocionais e compreensão teológica
- Suportar comparações de linhagens e variantes textuais
- Promover reflexão sobre fidelidade divina através das gerações

## 2. PÚBLICO-ALVO

### 2.1 Personas Primárias

**Iniciante Devocional** (25-40 anos)
- Necessidades: Conexões simples entre AT e NT
- Dores: Sobrecarga de informação textual
- Delícias: Insights rápidos e visuais

**Estudioso Casual** (30-50 anos)  
- Necessidades: Comparações entre variantes (MT vs LXX)
- Dores: Mapeamento manual entre fontes
- Delícias: Menu acadêmico com opções detalhadas

**Familiar** (35-55 anos)
- Necessidades: Interface mobile para grupos familiares
- Dores: Conteúdo muito técnico/seco
- Delícias: Timeline interativa e visual

### 2.2 Casos de Uso
1. **Estudo devocional pessoal**: Explorar linhagem durante leitura bíblica
2. **Ensino familiar**: Mostrar continuidade histórica para filhos
3. **Pesquisa acadêmica**: Comparar variantes textuais e cronologias
4. **Grupos de estudo**: Ferramenta visual para discussões

## 3. ESCOPO E FUNCIONALIDADES

### 3.1 In-Scope (MVP)

#### 3.1.1 Dados
- **62 gerações** organizadas em 4 blocos temáticos:
  - Pré-Dilúvio (Adão → Noé)
  - Pós-Dilúvio (Noé → Abraão)  
  - Monarquia (Abraão → Davi)
  - Exílio-Restauração (Davi → Jesus)
- **Fonte primária**: Bíblia NVT (Nova Versão Transformadora)
- **Metadados**: nomes, idades, pais/mães, referências bíblicas, notas

#### 3.1.2 Interface
- **Árvore interativa** com blocos coloridos por período
- **Menu dropdown** para seleções:
  - Versão bíblica (Mateus vs Lucas)
  - Idioma (PT-BR default)
  - Variante textual (MT vs LXX)
- **Tooltips** com versos bíblicos e significados
- **Timeline visual** com marcos históricos

#### 3.1.3 Experiência
- **Carregamento rápido** (<2s via Astro estático)
- **Mobile-first** responsivo
- **Acessibilidade** (ARIA, contraste adequado)
- **Temas visuais** (cores bíblicas: terra, azul, dourado)

### 3.2 Out-of-Scope (MVP)
- Backend/APIs dinâmicos
- Autenticação de usuários
- Recursos de áudio/vídeo
- Linhagens secundárias completas
- Cálculos cronológicos automáticos
- Funcionalidades de quiz/gamificação

### 3.3 Funcionalidades Futuras
- **Expansão de idiomas** (EN, ES)
- **Integração devocional** (planos de leitura)
- **Export PDF** para impressão
- **Modo offline** com PWA
- **Variantes adicionais** (Vulgata, Peshitta)

## 4. ESPECIFICAÇÕES TÉCNICAS

### 4.1 Arquitetura
**Stack**: Astro + Vue.js + TypeScript + JSON
- **Frontend**: Astro (páginas estáticas) + Vue islands (interatividade)
- **Dados**: JSON puro (`genealogia-completa-nvt.json`)
- **Build**: Vite + TypeScript
- **Deploy**: Cloudflare Pages
- **Sem backend**: 100% estático

### 4.2 Performance
- **Tamanho total**: <1MB (JSON ~5KB, assets leves)
- **TTI (Time to Interactive)**: <100ms (otimização Astro)
- **Compatibilidade**: Browsers modernos (ES6+)
- **Mobile**: Performance 95+ no Lighthouse

### 4.3 Estrutura de Dados
```typescript
interface GenealogyData {
  metadata: {
    title: string;
    version: string;
    source: string;
    variant: 'MT' | 'LXX';
    total_generations: number;
  };
  blocks: GenealogyBlock[];
  variants: Record<string, VariantInfo>;
}

interface GenealogyBlock {
  block_name: string;
  description: string;
  color_theme: string;
  generations: Person[];
}

interface Person {
  name: string;
  generation: number;
  father?: string;
  mother?: string;
  age_at_death?: number;
  biblical_reference: string;
  notes?: string;
}
```

## 5. EXPERIÊNCIA DO USUÁRIO

### 5.1 Jornada do Usuário
1. **Landing**: Citação acolhedora (Gn 5:1)
2. **Exploração**: Navegação intuitiva na árvore
3. **Descoberta**: Tooltips revelam detalhes bíblicos
4. **Personalização**: Menu permite customização
5. **Reflexão**: Notas teológicas inspiram devoção
6. **Saída**: Links para recursos devocionais

### 5.2 Princípios de UX
- **Simplicidade**: Interface limpa, hierarquia visual clara
- **Serenidade**: Paleta calma (tons terrosos/bíblicos)
- **Acessibilidade**: ARIA completo, navegação por teclado
- **Responsividade**: Mobile-first design

### 5.3 Interações Principais
- **Hover**: Revela tooltip com versículo e contexto
- **Click**: Expande detalhes da pessoa (modal/sidebar)
- **Menu**: Altera visualização (Mateus ↔ Lucas, MT ↔ LXX)
- **Busca**: Localiza pessoa específica na árvore
- **Zoom**: Navega em seções da timeline

## 8. RISCOS E MITIGAÇÕES

### 8.1 Riscos Técnicos
**Risco**: Performance com 62 elementos na árvore
**Mitigação**: Lazy loading, virtualização de componentes

**Risco**: Complexidade da visualização hierárquica
**Mitigação**: Usar bibliotecas testadas (Vue Flow, D3.js)

### 8.2 Riscos de UX
**Risco**: Sobrecarga de informação
**Mitigação**: Progressive disclosure, tooltips opcionais

**Risco**: Confusão entre variantes (Mateus vs Lucas)
**Mitigação**: UI clara, labels explicativos

### 8.3 Riscos de Conteúdo
**Risco**: Precisão das referências bíblicas
**Mitigação**: Revisão com especialistas, fontes múltiplas

**Risco**: Sensibilidades teológicas
**Mitigação**: Foco nos dados bíblicos, evitar interpretações controversas
